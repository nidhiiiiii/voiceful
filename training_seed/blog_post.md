# the moat is not the model

every "AI assistant" startup pitch in 2025 has the same shape. nice UI, decent model, some prompt engineering, a chat interface. they all do the same thing. they all sound the same.

the bit nobody is talking about: the model is commoditized. you can swap llama for qwen for mistral and your output quality moves a few percent. that's it. the model is a dial, not a moat.

the moat is context. who has the best signal of what the user is actually doing, right now, in this moment.

i built a thing this weekend that watches my git commits and drafts tweets in my voice. the model is small and open. the magic is that it sees what i'm doing without me having to tell it. by the time i'd opened twitter to write a post, the post was already drafted, in my voice, with the right hook, with my actual phrasing.

the model didn't do that. the watching did.

if you're building agents and you're spending all your time tuning prompts, you're playing the wrong game. spend it on the substrate. the watcher. the signal extractor. the part that knows what the user just did.

that's the moat.
