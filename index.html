04 — The Moat: Voice Profile
built from real artifacts, not prompt descriptions
01
ingests your tweets, commits, READMEs, blog posts, notes
02
computes statistical features: casing rate, emoji rate, em-dash usage, sentence length p90
03
LLM extracts voice attributes: formality, humor style, directness, hedging rate
04
picks 12 few-shot examples that best represent your voice per platform
05
every edit you make to a draft feeds back into the profile

06 — Five Triggers
Ship
commit with "ship", "release", "v1", "done" — or >5 files changed
Stuck
same error 3x in 60 min. same file edited 5x without commit. same test run 3x.
Insight
new note >50 words. contains "TIL", "learned", "huh". new draft post detected.
Idle
>3 days since last post. surfaces recent unposted events from the queue.
Asked
user types /draft <topic> in Telegram. direct invocation, no inference needed.
Each trigger fires
ship → Twitter + LinkedIn
stuck → Twitter (ask for help)
insight → all platforms
idle → best pending draft
07 — Live Demo Flow
1
make a real git commit git commit -m "ship parrot demo"
2
parrot detects it within 60 seconds git watcher polls every 60s. no webhooks needed.
3
telegram pings with a draft tweet [Approve] [Edit] [Reject] [Skip]
4
edit the draft. tap approve. the edit is captured and feeds back into your voice profile.
5
text copied to clipboard you paste it into Twitter manually. parrot never posts.
6
same insight, different platforms Twitter: 280 chars. LinkedIn: 200 words. dev.to: technical, with code blocks.
08 — Hard Rules (enforced in code, not prompts)
✗
never auto-posts. the approve button copies text to clipboard. you paste. every time, no exceptions.
✗
never invents facts. every draft is grounded in an actual artifact (commit message, note text, diff). no signal = no draft.
✗
never sends your data anywhere without consent. you configure the LLM provider. no telemetry. no third-party calls without an API key you set.
✗
never sanitizes your voice. if you wrote "this bug was annoying as hell", the draft does not soften it to "this issue presented some challenges".
✗
never uses em-dashes. sanitized post-LLM. because em-dashes are an AI tell and real builders don't write that way.
